Merge Projects

  • This workflow is meant to be used to merge a draft or a submitted project into existing one, in case the customer (or we) forgot to do so.
  • Avoid selecting project that is already sequenced as the source project.
  • Generally, older (smaller ID or larger status) project should be the destination project.
  • All samples from the source project will be merged into the destination project.
  • Samples with same name will be merged into same sample of the destination project.
  • After the merge, the source project will be empty and can be deleted manually.
  • If there are any attribute conflicts between samples, e.g. different values for the same attribute in same sample, they will need to be resolved manually in the project's Attributes-tab.
  • Sample qubit and fragment size will be the destination sample's values when merging projects. If destination sample's values are not measured, the source sample's values will be used if available.
  • Sample status and BA report ID will be the destination sample's values when merging projects. If destination sample's values are not available, the source sample's values will be used if available.
