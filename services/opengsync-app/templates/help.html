About

This program is designed to store and facilitate the transfer of information between customers and our team. It is not intended for designing or validating experimental setups. Users must ensure that their experimental setups are valid. If you are uncertain about your setup, please contact us at {{ contact_email }}. Each sequencing request is subject to review and approval by the our team. For non-standard experimental setups, such as using a custom genome reference, please contact us to ensure we can accommodate your request.

Data Structure Definitions


Sequencing Request

Customers can submit a sequencing request. The request defines samples, libraries, and pools that the customer wants to sequence. Users can submit two types of requests: raw samples library preparation + sequencing and ready-to-sequence pooled libraries only sequencing. Each sequencing request can be associated with a single group, e.g. research lab or a group of collaborators. You can create a new group in Groups-page. Results from each request will be shared with a single entity, whether it is a research lab or an individual and their optional collaborators. If you are submitting a request on behalf of multiple entities, e.g. two non-related projects from different groups, please submit separate requests for each entity. Results from different requests, including previous ones e.g. re-sequencing, can be grouped together for analysis by associating the samples with the same project. This is achieved by selecting an existing project and defining sample with a same 'sample_name' from existing project during sample/library annotation.

Project

Project is a collection of samples. Samples from same project are analysed and shared together. Project can include samples from different sequencing requests.

Sample

Sample biosample/specimen is a biological material that is prepared into a library and sequenced. Samples can contain metadata information such as tissue, sex, genotype etc. Each sample can be prepared into one or more libraries.

Library

Library is a sample that has been prepared for sequencing. Libraries are pooled before sequencing. Each library has a type that is defined by the library preparation workflow. Each library is associated to a single sequencing request. Sequenced bases, fastq-files, are shared with recipients indicated in request's 'share'-tab. Library can only be pooled and sequenced once. If you need to re-sequence a sample, new library will be created from that sample. Samples can be multiplexed in a same library, e.g. using 10X sample multiplexing hash-tag-oligos (HTOs).

Pool

Pool is a collection of libraries that are sequenced together. A pool can be sequenced on multiple lanes of a flowcell. Pool can be associated to only a single experiment; on re-sequencing, pool and libraries are cloned.
